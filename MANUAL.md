# AGSP Operation Manual

参謀本部方式を用いた実際の開発ワークフローです。

## 準備 (Setup)
1. **参謀本部の設営**:
   - Google Drive等にプロジェクト用フォルダを作成し、初期の設計思想、要件定義書（Markdown）を配置。
   - NotebookLMに上記フォルダをソースとして読み込ませる。
   - Gemini (Gems) または ChatGPT に「参謀長」プロンプトを設定。
2. **現場の設営**:
   - Claude Projects または Cursor に「現場指揮官」プロンプトを設定。
   - 知識ベースに初期仕様書をアップロード。

## 開発ループ (The Loop)

### Phase 1: 戦略策定 (Strategy)
1. 人間がNotebookLM/参謀長に「こういう機能が欲しい」と相談。
2. 参謀長が「仕様書ドラフト」を出力。
3. 人間が内容を確認し、Drive上の仕様書を更新（または新規作成）。

### Phase 2: 実装指令 (Deployment)
1. 現場指揮官（Claude）に「仕様書 Ver X.X に基づき実装せよ」と指示。
2. AIがコーディング、テスト実行。

### Phase 3: 逆流同期 (Reverse Sync / Error Handling)
**⚠️ 最も重要なフェーズです。エラーが出た時に「その場で直す」誘惑に打ち勝ってください。**

1. **Stop**: エラー発生。Claudeが勝手に直そうとするのを止める。
2. **Report**: エラーログをコピーし、参謀本部（NotebookLM/Gemini）へ持ち帰る。
3. **Audit**: 参謀長に問う。「このエラーが出た。仕様書のどこが間違っていたか？ 修正案を出せ」。
4. **Update**: 参謀長の提案に基づき、Drive上の仕様書を修正する。
5. **Re-Deploy**: 更新された仕様書を現場指揮官に再読込させ、「再実装」を命じる。
