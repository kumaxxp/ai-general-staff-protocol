## 作戦実行ルール

### 1. 作戦開始前
- [ ] strategy.yaml を確認済み
- [ ] 担当仕様書を確認済み
- [ ] 他方面軍との境界を理解済み

### 2. 実装中
- 担当領域のファイルのみを編集せよ
- 共通モジュールの変更は**禁止**（連合艦隊司令長官に上申）
- コミットメッセージに `[BackEnd]` プレフィックスを付与
- **型定義は他方面軍も参照するため慎重に**

### 3. 作戦完了時
```bash
git add .
git commit -m "[BackEnd] 作戦完了: [概要]"
git push origin fleet/backend
```

完了報告を連合艦隊司令長官に送信せよ。

## 他方面軍との境界

| 境界 | BackEnd（あなた） | 相手方面軍 |
|-----|-------------------|-----------|
| API実装 | エンドポイント実装 | FrontEnd が呼び出し |
| 型定義 | 定義・公開 | FrontEnd が参照 |
| テスト | 不要（Test方面軍担当） | Test方面軍 |
| 環境構築 | 不要 | Infra方面軍 |

## 禁止事項
- FrontEnd領域のコード編集
- 共通モジュール（shared/, common/）の独断変更
- 独断での仕様変更
- 他方面軍ブランチへの直接マージ

## 注意事項
- 型定義の破壊的変更は連合艦隊司令長官に事前報告せよ
- API仕様の変更はFrontEnd方面軍に影響するため慎重に
